# 研究課題10.01 
## 課題内容
**>>の意味** >>記法は、シェルに対して出力をファイルに追加することを指示する。
このときシェルは追加モード（第５章参照）を使うか、それとも単純にファイルの末尾にシークしてそこで書き込みを始めるか。シェルスクリプトを使った実験を考え出して、答えなさい。

## >>記法で記載するとリダイレクト先のファイルの末尾にデータを追加できる。

test sentencesとだけ書いてあるファイル、testtextに
who >> testtextのようなコマンドを打つと以下のようになる。

```
$ who >> testtext
$ cat testtext
test sentences
taichi84 seat0        2026-03-02 22:57 (login screen)
taichi84 :1           2026-03-02 22:57 (:1)
```

test sentencesという文のあとにwhoの結果が追加されている。
これは追加モードなのか、それとも単純にファイルの末尾にシークしてそこで書き込みを始めるかを実験したい。
予想としては、追加モードでの書き込みはアトミックな処理にとなるので、安全なため追加モードで書き込みしているはずだ。

## どういう実験をすればいいのか？
シークしてそこで書き込みを始めるのか、追加モードを使っているかはどうすれば判断できるか？

追加モードによる追加の特徴は、ファイルの末尾に移動して書き込みを始めるという処理はwrite時に同時に行われる。
これはアトミックな処理のため、このシークと書き込みの間に割り込むことはできない。

シークと書き込みを別々に行っているなら、この処理の間に割り込みが可能なはずである。
これをシェルスクリプトで割り込むようなことができれば、シークして書き込みを行っており、できなければ追加モードで追加しているということになる。

## シェルスクリプトによる実験で判断できるか？
同時に複数のプロセスに同じファイルに書き込みさせる場合も、究極的には予測の域をでないものと思われる。

strace -f -e openat で　straceしたものの子プロセスも追跡するようにし、openatしてるところだけを見れるようにする。
bash -c "who >> testtext"でstraceでbashの動きを確認するようにして追跡してみたら上記のようになった。

```
[pid 109282] openat(AT_FDCWD, "testtext", O_WRONLY|O_CREAT|O_APPEND, 0666) = 3
```
この行でtesttextを開くときにO_APPENDしているのがわかる。
つまりこれはseekしてwriteしているのではなく、open時に追加モードで開いてwriteしていることがわかった。

# 研究課題10.16
## 課題内容
**isatty** 標準出力に書き込むプログラムは、ファイルデスクリプタの先が端末かディスクファイル化について通常は意識しない。
本文では、ファイルディスクリプタの先がどうなっているかをプロセスから知る手段は内科のような話になっていたが、実はこれは誤りである。
ライブラリ関数のisatty(fd)は、fdが端末を参照している場合には真を返す。isattyはfstatシステムコールを使う。
fstatについて調べ、それに基づいて引数が通常のファイルに接続されたファイルディスクリプタなら真を返すisaregfile関数を書きなさい。

## 設計案
単純にfdをfstatして st_modeの項目にマスクをかけて、ファイルの種類を割り出せば良い。
割り出し方はmanページに記載があり、data.st_mode & S_IFMTでマスクをかけて、その値がS_IFREGにと等値になっているのを確認すれば良い)
```
if ((data.st_mode & S_IFMT) == S_IFREG) {
    return true;
  } else {
    return false;
  }
```

### コード 
```
#include <fcntl.h>
#include <stdio.h>
#include <stdlib.h>
#include <sys/stat.h>
#include <unistd.h>

int isaregfile(int fd) {
  struct stat data;

  if (fstat(fd, &data) == -1) {
    return -1;
  }
  return S_ISREG(data.st_mode);
}

int main(int ac, char *av[]) {
  int fd;

  if (ac != 2) {
    fprintf(stderr, "usage : isaregfile filename");
    exit(1);
  }

  fd = open(av[1], O_CREAT | O_TRUNC | O_RDWR, 0644);
  if (fd < 0) {
    perror("open");
    exit(1);
  }

  printf("開いたファイル、%sをチェックします。\n", av[1]);

  int rv = isaregfile(fd);

  if (rv == 1) {
    printf("通常のファイルに接続されています。\n");
  } else if (rv == 0) {
    printf("通常のファイル以外のファイルに接続されています。\n");
  } else if (rv == -1) {
    perror("fstat");
    exit(1);
  }

  close(fd);

  printf("標準出力をチェックします。\n");

  rv = isaregfile(STDOUT_FILENO);

  if (rv == 1) {
    printf("通常のファイルに接続されています。\n");
  } else if (rv == 0) {
    printf("通常のファイル以外のファイルに接続されています。\n");
  } else if (rv == -1) {
    perror("fstat");
    exit(1);
  }
}
```
int isaregfile(int fd)関数がファイルの種類を判定する関数。
st_modeをS_ISREGマクロでそのファイルが通常のファイルかをチェックすれば良い。


### 実行結果
```
$ ./isaregfile testfile

開いたファイル、testfileをチェックします。
通常のファイルに接続されています。
標準出力をチェックします。
通常のファイル以外のファイルに接続されています。

```

## まとめ
ファイルディスクリプタの接続先はfstatで接続先のファイルの情報をstat構造体を取得することで、
接続先が端末かファイルを簡単に調べることができる。
プロセス側からファイルディスクリプタの接続先を調べることは可能ということになる。

isattyもこれと同様な方法で接続先が端末かどうかを判定している。

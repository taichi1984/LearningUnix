# 研究課題10.13
## 課題内容
pipe.c を拡張して、３つの要素をもつパイプラインを処理できるようにしなさい。
この新バージョンは、引数として、３個のプログラム名を受付、それらをパイプラインとして実行するものとする。
例えば次のコマンドは、

pipe3 who sort head

who | sort | headと同じ効果になるようにする。

## 実際に作る
### 設計案
forkで子を３つ作り、子同士をpipeで接続するようにする。
子を作り、execして脱出させてから、２つ目のforkを作るようにする。
先にexecで脱出する前にforkをすると、最初にforkしたプロセスと親プロセスが２個目のforkをどちらも行ってしまうため、
プロセスが枝分かれしながら増えてしまうので、処理の順番に注意がいる。


### コード
```
/* pipe3.c
 * 目的：1つのプロセスから別のプロセスにデータを流す。
 * 　　　流されたプロセスはさらに別のプロセスにデータを流す。
 * パイプラインの作り方を示す。引数として２つのコマンドを取り、av[1]の出力をav[2]の出力に接続する。
 *
 *  使い方 pipe command1 command2 command3
 *  効果 : command 1 | command2 | command3
 *  制限 : コマンドは引数を取ることができない。
 *  引数の個数がわかっているのでexeclp()を使う。
 */

#include <stdio.h>
#include <stdlib.h>
#include <sys/wait.h>
#include <unistd.h>

#define oops(m, x)                                                             \
  {                                                                            \
    perror(m);                                                                 \
    exit(x);                                                                   \
  }

int main(int ac, char **av) {
  int thepipe1[2];
  int thepipe2[2];
  int newfd;
  int pid1;
  int pid2;
  int pid3;

  if (ac != 4) {
    fprintf(stderr, "usage:pipe cmd1 cmd2\n");
    exit(1);
  }
  if (pipe(thepipe1) == -1)
    oops("Cannot get a pipe", 1);

  if (pipe(thepipe2) == -1)
    oops("Cannot get a pipe", 1);

  /* ----------------------------------------------------------
   * パイプを作ったのでプロセスを２つにする */

  /* -----------------------------------------------------------
   * この時点でプロセスは２個ある。親はパイプから読み出す*/

  /* 子はav[1]を実行し、パイプに書き込む*/
  if ((pid1 = fork()) == 0) {
    if (dup2(thepipe1[1], 1) == -1)
      oops("could not redirect stdout", 4);

    close(thepipe1[0]); /*子はパイプから読み出さない*/
    close(thepipe2[0]);
    close(thepipe2[1]); // 子1はパイプ２を使わない
    close(thepipe1[1]); /*標準出力の複製を作り、パイプをクローズする。*/
    execlp(av[1], av[1], NULL);
    oops(av[1], 5);
  }

  /* 子はav[2]を実行し、パイプに書き込む*/
  if ((pid2 = fork()) == 0) {

    if (dup2(thepipe1[0], 0) == -1)
      oops("could not redirect stdout", 4);

    if (dup2(thepipe2[1], 1) == -1)
      oops("could not redirect stdout", 4);

    close(thepipe1[1]); /*子はパイプ１へ書き出さない*/
    close(thepipe2[0]); /*子はパイプ２から読み出さない*/
    close(thepipe1[0]); /*標準出力の複製を作り、パイプをクローズする。*/
    close(thepipe2[1]); /*標準入力の複製を作り、パイプをクローズする。*/

    execlp(av[2], av[2], NULL);
    oops(av[2], 5);
  }

  /* 子3はav[3]を実行し、パイプに書き込む*/
  if ((pid3 = fork()) == 0) {

    if (dup2(thepipe2[0], 0) == -1)
      oops("could not redirect stdout", 4);

    close(thepipe1[0]);
    close(thepipe1[1]); // 子３はpipe1は使わない。
    close(thepipe2[1]); /*子３はパイプ2へ書き出さない*/
    close(thepipe2[0]); /*標準入力の複製を作り、パイプをクローズする。*/
    execlp(av[3], av[3], NULL);
    oops(av[3], 5);
  }

  close(thepipe1[0]);
  close(thepipe1[1]);
  close(thepipe2[0]);
  close(thepipe2[1]);
  while (wait(NULL) > 0)
    ;

  return 0;
}
```

pipeの管理が若干ややこしい。pipeの書き込み側(pipeに書き込む側)をプロセスの標準出力につなぎ
pipeの読み込み側(pipeから読み込む側)をプロセスの標準入力につなごう。

pipeはバッファと考え、プロセスがファイル読み書きの主体と考えたときに、

pipeの読み出し用は pipe「から」読み出すためのFD
pipeの書き出し用は pipe「に」書き込むためのFD
プロセス側の標準入力はファイル「から」読み出すためのFD
プロセス側の標準出力はファイル「に」書き出すためのFD

であるということをしっかり認識しておこう。
標準入力、標準出力、読み出し用、書き出し用等の用語が混乱しやすいので注意。

dup2による接続先さえあっていれば、あとはpipeを全部クローズするだけなので、どこに何をつなぐかだけよく考えて
あとは使ったパイプは全部閉じるというのを実行していれば、パイプの接続は完成する。

親プロセスはwaitするだけだが、無駄なpipeは作られているので、しっかりcloseはしておく必要がある。

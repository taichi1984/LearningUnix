# 研究課題10.06
## 課題内容
**ユーザーの保護** ユーザーがsort <data >dataと入力したらどうするか。
この要求にはどのような問題があるか。Unixの標準シェルは、このような要求にどのように対処しているか。
smshはこの問題にどのように対処するか。

## 実際にやってみる

```
$ cat data
akllkjsaklj
lkjaklsjcklja
,lam,.sm,.amp@o
qlkjqkljq
sskjakljc
bnaal;klk;cks
paskll;sskjakljc
klaj;lkc;lam
$ sort <data >data
$ cat data

```

dataは何もないという内容に書き換えられてしまう。
これはどうして起こるのか？

### 何をしているのか？
これはsortの標準入力の入力元をdataにし、標準出力をdataにしている。
sortは標準入力からの入力をきちんと出力できるのか。

```

$ cat data
,lam,.sm,.amp@o
akllkjsaklj
bnaal;klk;cks
klaj;lkc;lam
lkjaklsjcklja
paskll;sskjakljc
qlkjqkljq
sskjakljc

$ cat data | sort -r
sskjakljc
qlkjqkljq
paskll;sskjakljc
lkjaklsjcklja
klaj;lkc;lam
bnaal;klk;cks
akllkjsaklj
,lam,.sm,.amp@o
```

sortは標準入力からの入力を受け付けている。

```
$ sort <data >data2
$ cat data2
,lam,.sm,.amp@o
akllkjsaklj
bnaal;klk;cks
klaj;lkc;lam
lkjaklsjcklja
paskll;sskjakljc
qlkjqkljq
sskjakljc

```
パイプであってもリダイレクトであってもきちんと標準入力からsortはデータを受取り標準出力に吐き出している。

## 標準入力と標準出力のリダイレクト先を同じにした場合、なぜデータは失われてしまうのか？
やっている事的には、このようなことをした場合、内容が変わらずリダイレクトされそうなものである。
ではなぜそうはならずに、データが消えてしまうということが起こるのか？

### 標準出力のリダイレクト時にはcreat()であらたなfdを作る
creat()関数はファイル名がない場合は新しく作り、ファイル名のファイルがある場合は、そのファイルの内容を消して新たに頭からファイルを書いていくことになる。
リダイレクトをするためにcreatした瞬間にファイルの内容は空になるため、空になったファイルからデータを読もうとしても、何も読み込むことができない。
データがないファイルから読み込みを行ってもデータはないので、書き込み先もデータがない状態になる。

リダイレクトするタイミングはsortが実行されるタイミングではなく、その前に行われるため、
読み込み先と書き込み先を同じ箇所にリダイレクトした場合、書き込み先のリダイレクト時にそのファイルの内容は失われ、sortが読み込もうとしたときにはファイルの内容は消失している。
故に、何も出力されないようなことが起こる。

## Unixの標準シェルはこのような要求にどう対処しているか。
bashでの実験によると、特に対処はしていないものと思われる。
実際に実験を行った場合、ファイルの内容は消えてしまった。

## smshはこの問題にどのように対処するか。
もし対処しようとするならば、構文分析してリダイレクトの処理を行うときにすべてのリダイレクト関連のものを抜き出して、
標準入力と標準出力のファイルが一致した時点でエラーを出すのが良いと思う。

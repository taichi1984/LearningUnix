# プログラミング課題11.10
## 課題内容
sortをサブルーチンとして使うプログラムを書きなさい。このプログラムは、まずデータ行を文字列配列に読み込み、
次に２つのパイプを作って、sortを実行するプロセスを作るものとする。そして、１本のパイプを使って入力行のシーケンスを
sortに送り込み、そのパイプをクローズする。さらに、もう１本のパイプを通じて、sortの結果を読み出し、配列に書き込む。
最後に配列を表示するようにしなさい。

## データの扱いについて
今回はデータの大きさはchar data[60000]と大きさを適当に決めることにする。
配列もsentences[255][255]と固定長にし、mallocなどの手間を省く。
本来、実務で実装する場合にはmallocなどを使って可変長配列にしたり、データの大きさも考える必要があるが、
今回の課題の本筋はそこではないと思われるので、上記のものとする。

### ソース
```
#include <stdio.h>
#include <stdlib.h>
#include <sys/wait.h>
#include <unistd.h>

int main(int ac, char *av[]) {

  char *filename;
  int toSort[2], fromSort[2];
  if (ac != 2) {
    fprintf(stderr, "invalid argument\n");
    exit(1);
  }

  filename = av[1];

  int ret = pipe(toSort);
  if (ret == -1) {
    perror("pipe");
    exit(1);
  }

  ret = pipe(fromSort);
  if (ret == -1) {
    perror("pipe");
    exit(1);
  }

  int pid = -1;
  if ((pid = fork()) == -1) {
    perror("fork");
    exit(1);
  }

  if (pid == 0) {
    // 子プロセスの処理(sort側）
    dup2(toSort[0], 0);
    dup2(fromSort[1], 1);

    close(toSort[0]);
    close(toSort[1]);
    close(fromSort[0]);
    close(fromSort[1]);

    char *args[] = {"sort", NULL};
    execvp("sort", args);
    perror("exec");
    exit(1);
  } else {

    // 親プロセスの処理

    FILE *fromSortfp = fdopen(fromSort[0], "r");

    close(toSort[0]);
    close(fromSort[1]);

    FILE *source_fp;
    char data[60000];

    source_fp = fopen(filename, "r");
    if (source_fp == NULL) {
      perror("fopen");
      exit(1);
    }

    FILE *toSortfp = fdopen(toSort[1], "w");
    if (toSortfp == NULL) {
      perror("fdopen");
      exit(1);
    }

    while (fgets(data, sizeof(data), source_fp)) {
      fputs(data, toSortfp);
    }

    fclose(source_fp);
    fclose(toSortfp);

    char sentences[255][255]; // 簡略化のため配列、文字列の長さは固定にする。
    int i = 0;
    while (fgets(sentences[i], sizeof(sentences[i]), fromSortfp)) {
      i++;
    }

    fclose(fromSortfp);

    wait(NULL);
    int num = i;

    for (int i = 0; i < num; i++) {
      printf("%s", sentences[i]);
    }
  }
}
```

## 実装した際に詰まったポイント
### pipeについて
pipeを作る際に、認識の甘さから以下の思い違いをしていた。

pipe()を読んで、forkしてdup2で接続先を決めてからfdを閉じてexecするとい一連の流れを丸暗記していたせいで、
dup2が接続する作業だと思いこんでいた部分があった。このため、標準出力と標準入力に付け替えたくない場合、どうすればいいのかがわからなくなっていた。

上記は大きな勘違いで、２つのプロセス間でパイプを接続するところは
pipe()を呼んだあとにfork()することにある。これで２つのプロセスから同じパイプにfdが繋がれているという形になるのだ。
不要なfdを閉じることで、pipeを通して２つのプロセス間ですでにやりとりは成立することになる。

dup2はその経路を複製して、指定したFD番号につなぎ直しているだけなので、２プロセス間を接続する作業ではない。
fdopenはすでに接続されているそのpipeのfdを便利に使うためにfpに変換しているだけである。
これが非常に重要。

### fcloseについて
readしている側のプロセスはwriteしている側のプロセスがcloseでEOFを送信することによって、readを終了すべきなのを認識する。
そのため、write側がfcloseをしない場合、read側はreadを終了することができず、write側もう書くデータがないのにも関わらず
EOFがくるまでreadをし続けようとしてしまうため、ブロッキングが解除されずが処理が止まってしまうことがある。
今回はwrite側のtoSortfpをfcloseするのを忘れたため、sortがreadのループから抜けられず、ソート処理及びwriteの処理まで到達しないことがあった。
fdは使い終わったらすぐ閉じるのを意識する必要がある。

## まとめ
pipeとforkによってプロセス間に通信経路を構築し、
dup2やfdopenを用いてその経路を任意のインターフェースに接続することで、
外部コマンドをサブルーチンのように扱うことができる。

この際、不要なファイルディスクリプタを適切にcloseし、
EOFを正しく伝播させることが重要である。

これらの仕組みを理解することで、UNIXにおけるI/Oの基本構造を把握することができる。

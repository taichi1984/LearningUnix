# 研究課題7.6
## 課題内容
**システムコールへの割り込み** Unixの一部のバージョンではgetch呼び出しはタイマチック処理時に割り込まれる。
このようなシステムでは、getchは、タイマチックが発生するたびにEOFを返す。
この動作は、プログラムにどのような影響を与えるか、それは問題か、変更することはできるかを答えなさい。

## この動作はEOFを利用して、割り込み完了後に行う処理を決めることができる。
メインループのほうで

if( (ch = getch()) == EOF)
    // 行いたい処理

とすることによって、割り込みが完了後に行う処理を設定することができる。
これを利用して、割り込みが発生したときのみ行う表示処理などを書くことが可能。
getch()は端末からの読み込みになるため、ファイルの終端は存在しない。
そのため、割り込み後の後処理のためにEOFを上手くりようすることができる。

## この処理を行わないようにするためには、どうすればいいか？
sigactionの設定を行えば良い。
sigaction.sa_flags |= SA_RESTART;

のようにすればフラグを設定すれば、read()あEINTRで中断される代わりに自動的に再開される。
その場合は、getch()はブロックをしつつけ、EOFを返さないようにすることができる。

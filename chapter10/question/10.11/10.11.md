# プログラミング課題10.11
## 課題内容
whotofile.cサンプルを書き換え、whoの出力をファイルに追加するようにしなさい。
ファイルが存在しなくても動作するようにすること。

## 現在のwhotofile.c
### 仕様
現在のwhotofile.cは、プログラムの出力をリダイレクトし、新たにuserlistというファイルを新しく作っている。
これを、新しく作るのではなく、既存のものに追加するように作れば良い。
### コード
```

/* whotofile.c
 * 目的: 他のプログラムの出力をリダイレクトするにはどうすればよいかを示す。
 * 考え方：forkしてから、子の中で出力をリダイレクトし、最後にexecする。
 */

#include <fcntl.h>
#include <stdio.h>
#include <stdlib.h>
#include <sys/wait.h>
#include <unistd.h>

int main() {
  int pid;
  int fd;

  printf("About to run who into a file\n");

  /* 新プロセスを作成する。失敗したら終了*/

  if ((pid = fork()) == -1) {
    perror("fork");
    exit(1);
  }
  if (pid == 0) {
    close(1);
    fd = creat("userlist", 0644);
    execlp("who", "who", NULL);
    perror("execlp");
    exit(1);
  }
  if (pid != 0) {
    wait(NULL);
    printf("Done running who. results in userlist\n");
  }
}

```

## 改良案
creatでファイルを作成しているのを、open("filename", O_WRONLY | O_APPEND | O_CREAT , 0644)でファイルを作成するようにすればよい。
ついでに書き込むファイルをコマンドライン引数で指定できるように修正する。

### コード
```

/* whotofile.c
 * 目的: 他のプログラムの出力をリダイレクトするにはどうすればよいかを示す。
 * 考え方：forkしてから、子の中で出力をリダイレクトし、最後にexecする。
 */

#include <fcntl.h>
#include <stdio.h>
#include <stdlib.h>
#include <sys/wait.h>
#include <unistd.h>

int main(int ad, char *av[]) {
  int pid;
  int fd;

  if (ad != 2) {
    fprintf(stderr, "usage : whotofile2 filename\n");
    exit(1);
  }

  printf("About to run who into a file\n");

  /* 新プロセスを作成する。失敗したら終了*/

  if ((pid = fork()) == -1) {
    perror("fork");
    exit(1);
  }
  if (pid == 0) {
    close(1);
    fd = open(av[1], O_WRONLY | O_APPEND | O_CREAT, 0644);
    execlp("who", "who", NULL);
    perror("execlp");
    exit(1);
  }
  if (pid != 0) {
    wait(NULL);
    printf("Done running who. results in %s\n", av[1]);
  }
}

```

コマンドライン引数で取るあたって、コマンドライン引数の個数チェックを行う処理を追加。
creatで行っていたファイルの作成処理を
```
    fd = open(av[1], O_WRONLY | O_APPEND | O_CREAT, 0644);
```

で行うように変更した。

## dup2について
openで開くファイルディスクリプタは、現在使われていないファイルディスクリプタの最も若い番号に適用されるので、
上記のプログラムは基本的には正しく動くが、本来はdup2を使い、ファイルディスクリプタを明示的に切り替えるのが望ましい。

### コード

```
/* whotofile.c
 * 目的: 他のプログラムの出力をリダイレクトするにはどうすればよいかを示す。
 * 考え方：forkしてから、子の中で出力をリダイレクトし、最後にexecする。
 */

#include <fcntl.h>
#include <stdio.h>
#include <stdlib.h>
#include <sys/wait.h>
#include <unistd.h>

int main(int ad, char *av[]) {
  int pid;
  int fd;
  if (ad != 2) {
    fprintf(stderr, "usage : whotofile2 filename\n");
    exit(1);
  }

  printf("About to run who into a file\n");

  /* 新プロセスを作成する。失敗したら終了*/

  if ((pid = fork()) == -1) {
    perror("fork");
    exit(1);
  }
  if (pid == 0) {
    fd = open(av[1], O_WRONLY | O_APPEND | O_CREAT, 0644);
    if (fd == -1) {
      perror("open");
      exit(1);
    }

    dup2(fd, 1);
    close(fd);
    execlp("who", "who", NULL);
    perror("execlp");
    exit(1);
  }
  if (pid != 0) {
    wait(NULL);
    printf("Done running who. results in %s\n", av[1]);
  }
}
```
dup2(fd,1)で、1に割り当てられていたfdを openで開いたfdと同じ対象に差し替える。
すでにfd=1が開かれている場合は、自動的にcloseされるため、明示的にclose(1)をする必要はない。
差し替えたあとはopenしたときに得たfdはもう不要のため、closeしている。

上記のコードをし、同様の結果を得られることを確認した。
dup2で適切にFDのつけかえが行われていることがわかった。

## まとめ
ファイルへの追記と、存在しない場合の作成は open のフラグ
(O_WRONLY | O_APPEND | O_CREAT) によって実現できる。

また、open は最小の未使用ファイルディスクリプタを返すため、
close(1) の後に open を呼ぶことで、標準出力（fd=1）をファイルに
割り当てることができる。

ただし、この方法はファイルディスクリプタの割り当て規則に依存するため、
より安全で明示的な方法として dup2(fd, 1) を用いるのが望ましい。

用途に応じて、暗黙的な方法（close + open）と明示的な方法（dup2）を
使い分けることが重要である。

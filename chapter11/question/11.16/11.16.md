# プログラミング課題11.16
## 課題内容
**キャッシュとプロキシ** 前問のプロキシサーバーの説明を読みなさい。
時刻は、１秒に１度ずつしか変わらないので、プロキシサーバーが数ミリ秒ごとに何度もサーバーを読み出しても無意味だ。
サーバーから読み出した時刻を格納し、その時刻が１秒以上古いものになるまで、新しいサーバー呼び出しを行わないようにする
時報サーバーのキャッシングプロクシを書きなさい。


## プロキシサーバーを確認

int setup_server(int port) {
  struct sockaddr_in saddr;
  char hostname[HOSTLEN];
  int sock_id;

    return -1;

    return -1;

  /*
   * ステップ3:キューサイズを１としてソケットを着信できる状態にする。
   */

  if (listen(sock_id, 5) != 0)
    return -1;

  return sock_id;
}

ssize_t write_all(int sock_fd, char *buf, int rn) {
  int wr = -1;
  int total = 0;
  while (total < rn) {
    wr = write(sock_fd, buf + total, rn - total);

    if (wr <= 0) {
      return -1;
    }
    total += wr;
  }
  return total;
}

## まとめと注意点
キャッシュはキャッシュできちんと変数を用意して格納しておく必要がある。
直接ストリームとして送る場合はreadしたものをwriteしていけば、readしたところが全文含まれてない場合もreadした分だけwriteすれば成立するが
cacheに保存していく場合は、読んだ分だけcacheにコピーしていくようにしないと、齟齬が出る可能性がある。
cacheの長さをcache_lenで管理した上で、cacheを出力したい場合は write(fd,cache,cache_len)という形で出力するようにしよう。
今回はメッセージの量がBUFSIZに満たないことが明確なので、cache[BUFSIZ]で宣言したが、
メッセージの量が大きい場合は、mallocを使って動的に長さを取得する必要が出てくることもある。

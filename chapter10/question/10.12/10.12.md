# 研究課題10.12
## 課題内容
sortコマンドの入力をリダイレクトして、ファイルから読み出せるようにするsortfromfile.cというプログラムを書きなさい。
入力ファイル名は、コマンド行で指定するものとする。

## 方針
forkして、forkしたプロセスの標準入力のファイルディスクリプタをdup2で読み込む対象のファイルにつけかえる。
そしてsortを実行するだけ。

### コード

```
/* sortfromfile.c
 * 目的: sortの入力をリダイレクトし、ファイルから読み込めるようにする。
 * 考え方：forkしてから、子の中で入力をリダイレクトし、最後にexecする。
 */

#include <fcntl.h>
#include <stdio.h>
#include <stdlib.h>
#include <sys/wait.h>
#include <unistd.h>

int main(int ad, char *av[]) {
  int pid;
  int fd;

  if (ad != 2) {
    fprintf(stderr, "usage : sortfromfile filename\n");
    exit(1);
  }

  /* 新プロセスを作成する。失敗したら終了*/

  if ((pid = fork()) == -1) {
    perror("fork");
    exit(1);
  }
  if (pid == 0) {
    fd = open(av[1], O_RDONLY);
    if (fd == -1) {
      perror("open");
      exit(1);
    }

    dup2(fd, 0);
    close(fd);
    execlp("sort", "sort", NULL);
    perror("execlp");
    exit(1);
  }
  if (pid != 0) {
    wait(NULL);
  }
}
```

標準入力のつけかえをしてからsortを実行しただけ。
これで正しく動作する。

## 実務での話について
そもそもsortは引数に入力元のファイルを取ることができる。
実務的にみれば、これは無駄なdup2をしているので実務的には意味のないプログラムである。
今回はdup2での標準入力のつけかえの理解のためのプログラムだと思われる。

## まとめ
標準出力と同様に標準入力もdup2での付け替えを行うことができる。
今回の例は学習用のものであったが、自分でツールを作る際は重要になってくるのでしっかり覚えておくこと。

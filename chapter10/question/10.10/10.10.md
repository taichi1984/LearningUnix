# 研究課題10.10
## 課題内容
watch.shにもっとよい機能を追加しよう。

(a)このバージョンは、すべてのユーザーのログイン、ログアウトを出力する。
しかし、コマンド行引数として、監視対象のユーザーのリストを含むファイル名を受け付けるようにしたほうが役に立つだろう。

(b)このバージョンは、変化がなくてもループを実行するたびに何かしら出力をする。
なにか表示すべき情報があるときに限り、new logins , new logoutsメッセージを出力するように、プログラムを書き換えなさい。

(c)whoコマンドは、ユーザー名のほか、ログイン時刻や端末名を表示する。しかし、これでは必要以上に情報が出力されてしまう。
ユーザーが第２ウィンドウを使って接続していても、それはどうでもよいことだ。
端末の違いに関わらず、監視対象のユーザーが"ログオンしている"状態から、"ログオンしていない"状態に変わったタイミングを報告するようなプログラムを書きなさい。

(d)このバージョンは、データをカレントディレクトリのprev,currファイルに格納するが、
プログラムが終了しても、これらのファイルを残したままである。このような設計には、いくつかの理由で問題がある。
どのような理由が考えられるだろうか。スクリプトが一時ファイルを使い、終了時にそれらのファイルを削除するように書き換えなさい。
シェルのtrapコマンドについて、またmktempユーティリティの使い方について調べなさい。

## watch.shを確認する。
```
who | sort > prev
while true ; do
  sleep 10
  who | sort > curr
  echo "logged out:"
  comm -23 prev curr
  echo "logged in:"
  comm -13 prev curr
  mv curr prev

```

## (a)の機能追加について
### 調査項目と調査結果
１、シェルスクリプトはコマンド行引数をどうやって受け付けるのか。
シェルスクリプトはコマンド行引数で受け付けた引数を順番に$1,$2という順番で変数に格納していくようになっている。

２、どうすれば監視対象のユーザーのみ、ログイン、ログアウトを出力するようにできるのか。

監視対象のユーザーリスト、ログアウトしたユーザーのログファイル、ログインしたユーザーのログファイルを作成し
監視対象のユーザーリストと、ログアウトしたユーザーのログファイルの１フィールド目を比較して、一致していたら出力する。


### コード
```
who | sort > prev
while true ; do
  sleep 10
  who | sort > curr
  comm -23 prev curr > out_log
  comm -13 prev curr > in_log
  echo "logout : "
  awk 'NR==FNR { users[$1]; next }  $1 in users' "$1" out_log
    
  echo "login : "
    awk 'NR==FNR { users[$1]; next } $1 in users' "$1" in_log
  mv curr prev
  rm out_log in_log
done
```

### 実行結果

$ ./watch2.sh

login : 
taichi84 pts/2        2026-03-25 01:08 (127.0.0.1)
logout : 
taichi84 pts/2        2026-03-25 01:08 (127.0.0.1)
login : 

### このコードの注意点
awkの機能の変数の$1とshellのコマンドライン引数を格納する変数である$1の混同に気をつける。
常にawkのコマンドを書く''のなかの表記か、shellとしての表記なのかを意識してコードを見る必要がある。


## (b)の機能追加について
### 調査項目
１、prev currの差分を検出し、差分がない場合はechoの処理を行わないようにするためにはどうすればよいか？

echo表示の処理の前に、prevとcurrに差分があるかどうかを比較するコマンドを挟めば良い。
それにはcmpが適したコマンドである。

```
cmp -s prev curr
```

上のコマンドはprev と currを比較し、入力ファイルの内容が同じであれば0、違えば1、問題が発生した場合は2を返す。
'-s'オプションは出力をさせないためのオプション。終了結果のみを使い、画面への出力は捨てたいのでこれを使う。
これでファイルの中身が違っていた場合のみ、表示することにしよう。


### コード
```
who | sort > prev
while true ; do
  sleep 10
  who | sort > curr
  if ! cmp -s prev curr
  then
  comm -23 prev curr > out_log
  comm -13 prev curr > in_log
  echo "### logout ### "
  awk 'NR==FNR { users[$1]; next }  $1 in users' "$1" out_log
  echo ""    
  echo "### login ### "
    awk 'NR==FNR { users[$1]; next } $1 in users' "$1" in_log
  echo ""
  rm out_log in_log
  fi
  mv curr prev


done
```
このコードで、差分があったときのみ表示されるようになった。

### 実装時の注意点
そもそもcmpでprevとcurrの差分があるかの確認をするため、差分がなかった場合は、out_logやin_logを作る必要がない。
そのため、if ! cmp -s prev currはcommの比較前におき、 out_log in_logの削除は fiの前に行う必要がある。


## (c)の機能追加について
### 調査項目
ユーザーが１つ以上のウィンドウでログインしているのをどう確認すればよいのか？
１つもログインしていない場合はどう判断すればいいか？

### 問題文の確認と設計計画
問題文は以下のものになる。
```
(c)whoコマンドは、ユーザー名のほか、ログイン時刻や端末名を表示する。しかし、これでは必要以上に情報が出力されてしまう。
ユーザーが第２ウィンドウを使って接続していても、それはどうでもよいことだ。
端末の違いに関わらず、監視対象のユーザーが"ログオンしている"状態から、"ログオンしていない"状態に変わったタイミングを報告するようなプログラムを書きなさい。
```
このプログラムの趣旨はログインしたユーザーとログアウトしたユーザーを通知することである。
問題文からはログイン時刻や端末名の表示は必要ないと読み取れる。
このことから考えるべきは、whoの中にユーザー名が１個以上あるか？１個もないか？を確認すれば良い。

whoで出力した内容の１フィールド目はユーザー名である。
このユーザー名をawkで切り出し、uniqで重複削除した上で、prevとcurrに書き込むことにより、
今ログインしているユーザー(１つ以上の端末でログインしているユーザー)を記録できる。（２つ以上のウィンドウでログインしていても、それは重複削除されて１つになる）
あとは差分をcommで比較すれば、ログインしたユーザーとログアウトしたユーザーを正しく表示できる。

### コード
```
who | sort | awk '{print $1}' | uniq  > prev
while true ; do
  sleep 10
  who | sort | awk '{print $1}' | uniq > curr
  if ! cmp -s prev curr
  then
  comm -23 prev curr > out_log
  comm -13 prev curr > in_log
  echo "### logout ### "
  awk 'NR==FNR { users[$1]; next }  $1 in users' "$1" out_log
  echo ""    
  echo "### login ### "
    awk 'NR==FNR { users[$1]; next } $1 in users' "$1" in_log
  echo ""
  rm out_log in_log
  fi
  mv curr prev


done
```

### 実行結果
```
---bobでログイン---
### logout ### 

### login ### 
bob

---bob(２つ目)でログイン---

---bob(１つ目)をログアウト---

---bob(２つ目)をログアウト---
### logout ### 
bob

### login ### 

```
特定のユーザーが１つ以上のウィンドウでログインしたときに、ログインを通知し
全てのウィンドウを閉じたときにログアウトを通知するようになった。

## (d)の機能追加について
１，ファイルを残したままだとどういった不都合が発生するか。
２，終了する方法をどのように追加するか。
３，シグナルで停止したとき、シェルでもシグナルハンドラ的なものを使って、残ったファイルを削除する後処理は可能なのか。

### 1への解答
ファイルを残したままの場合、他のプログラムが同じ名前のtmpファイルを作るような仕様だったりすると、起動時に競合する恐れがある。
また、このプログラムはログイン、ログアウトを通知するためのプログラムのため、ユーザーの意図と違う形でファイルが残されるのはよくない。

### 2への解答
終了する方法は今回は特にコマンドから終了するような作りにはなっていない。
そのため、SIGINTやSIGTERMなどのシグナルで終了する仕様とする。

### 3への解答
trapコマンドでシグナルハンドラを設定できる。

### コード
```
cleanup(){
  [ -e prev ] && rm prev
  [ -e curr ] && rm curr
}
trap cleanup EXIT

who | sort | awk '{print $1}' | uniq  > prev
while true ; do
  sleep 10
  who | sort | awk '{print $1}' | uniq > curr
  if ! cmp -s prev curr
  then
  comm -23 prev curr > out_log
  comm -13 prev curr > in_log
  echo "### logout ### "
  awk 'NR==FNR { users[$1]; next }  $1 in users' "$1" out_log
  echo ""    
  echo "### login ### "
    awk 'NR==FNR { users[$1]; next } $1 in users' "$1" in_log
  echo ""
  rm out_log in_log
  fi
  mv curr prev


done
```

cleanup関数でシグナルハンドラを作り、trapでハンドラを設定する。
EXIT時にハンドラを動かすようにすれば、SIGINTやSIGTERM、その他の終了時にはprev,currファイルの有無を確認し削除してくれる。

### mktempとは？
現時点では自前でprev currなどの一時ファイルを作っているが、これは他のプロセスとの競合などのリスクがある。
mktempを使うことで、安全に一時ファイルを作ることが可能。ファイルの存在チェックと、そのファイル名がなければ作ることをアトミックな処理で行ってくれる。
mktempを使い、安全に一時ファイルを作って実装したコードが以下のものになる。

### コード
```
prev=$(mktemp)
curr=$(mktemp)


cleanup(){
  [ -e "$prev" ] && rm "$prev"
  [ -e "$curr" ] && rm "$curr"
}
trap cleanup EXIT

who | sort | awk '{print $1}' | uniq  > "$prev"
while true ; do
  sleep 10
  who | sort | awk '{print $1}' | uniq > "$curr"
  if ! cmp -s "$prev" "$curr"
  then
  comm -23 "$prev" "$curr" > out_log
  comm -13 "$prev" "$curr" > in_log
  echo "### logout ### "
  awk 'NR==FNR { users[$1]; next }  $1 in users' "$1" out_log
  echo ""    
  echo "### login ### "
    awk 'NR==FNR { users[$1]; next } $1 in users' "$1" in_log
  echo ""
  rm -f out_log in_log
  fi
  cp "$curr" "$prev"


done
```
tempファイルを作って、それをprev変数とcurr変数に代入することにより、
安全な一次ファイルを使ってこのシェルスクリプトを実行することができた。
一時ファイルを使う場合は、mktempを使って作るようにすること。

### その他細かいバグ修正後の最終バージョン
```
prev=$(mktemp)
curr=$(mktemp)
out_log=$(mktemp)
in_log=$(mktemp)


cleanup(){
  rm -f "$prev" "$curr" "$out_log" "$in_log"
}
trap cleanup EXIT

who | awk '{print $1}' | sort -u > "$prev"
while true ; do
  sleep 10
  who | awk '{print $1}' | sort -u > "$curr"
  if ! cmp -s "$prev" "$curr"
  then
  comm -23 "$prev" "$curr" > "$out_log"
  comm -13 "$prev" "$curr" > "$in_log"
  echo "### logout ### "
  awk 'NR==FNR { users[$1]; next }  $1 in users' "$1" "$out_log"
  echo ""    
  echo "### login ### "
    awk 'NR==FNR { users[$1]; next } $1 in users' "$1" "$in_log"
  echo ""
  : > "$out_log" 
  : > "$in_log"
  fi
  cp "$curr" "$prev"
done
```

out_log、in_logをmktempを使うようにする。
mktempを使った場合、ファイルを削除せず、中身だけの削除にして、名前を残すようにする。
cleanupハンドラで、out_log in_logも削除する
prev currの作成時、sortの前にawkで先に１個目のフィールドを切り出すように変更。
sort | uniq を sort -uに変更。
必要なクォートを追加するなど行いました。

# 研究課題11.19
## 課題内容
**popenの使い方** Cプログラマなら、argv[0]には、通常、起動されたプログラムの名前が格納されていることを知っている。
しかし、プロセスには、少々遠回りながら、自分の名前を取得できるもう１つの方法がある。
popenを使ってpsコマンドの出力を取得し、自分自信のプロセスID番号を探すのである。
この方法を使うプログラムを書きなさい。

## コード
```
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

int main() {
  FILE *pfp = NULL;
  int mypid = getpid();
  char cmd[BUFSIZ];
  char filename[BUFSIZ];
  snprintf(cmd, sizeof(cmd), "ps -p %d -o comm=", mypid);
  // snprintf(cmd, sizeof(cmd), "ps | awk '{if ($1 == %d){print $4}}'", mypid);
  // // awk版。psの実装によって安定しないので非推奨

  if ((pfp = popen(cmd, "r")) == NULL) {
    perror("popen");
    exit(1);
  }

  int c;

  while ((c = getc(pfp)) != EOF) {
    printf("%c", c);
  }
  pclose(pfp);
  return 0;
}
``` 
## 実行結果
```
$ gcc -o getname getname.c
$ ./getname
getname

```

## まとめ
popenの内容に変数を使いたい場合は、一度snprintfを使って、変換後の文字列を用意してからpopenに渡すこと。
今回は最初はawkで実装したが、ChatGPTで確認したところ、psの実装は環境ごとに異なる場合があるとのこと。
基本は各コマンド＋オプションで実行できることならコマンド+オプションで実行するのが安全度は高いだろう。
もちろん、現環境における対象コマンドの仕様をきちんと把握した上で、awkで切り分けるならそれも構わないが、
パイプなど、無駄にリソースを消費する行為ではあるので、基本的にはコマンドでできることはコマンドで行おう。

今回はpsコマンドの -pオプションでどのpidの内容を表示するか指定し、-o comm=を指定したことでコマンド名だけ表示するように指定している。

現状では、accept(sock_id, NULL , NULL)となっているので、この第２引数と第３引数に構造体などを渡して貰うのが情報を受けるよくある仕組みである。

## acceptのmanページを確認する。
### 書式の確認
まずは書式を確認し、引数は何が入るのかを確認する。
```
int accept(int sockfd, struct sockaddr *addr, socklen_t *addrlen);
```
第２引数にはsockaddr ,　第３引数にはsocklen_tが入るようだ。
sockaddrはbindするときにも使った、bindに必要なアドレスなどを格納する構造体である。
sockaddrの内容を確認してみよう。

### sockaddrの確認
```
struct sockaddr_in {
           sa_family_t     sin_family;     /* AF_INET */
           in_port_t       sin_port;       /* Port number */
           struct in_addr  sin_addr;       /* IPv4 address */
       };
```

インターネットで利用されるsockaddrはsockaddr_in か sockaddr_in6となっている。sockaddr_in6はIPv6対応。
今回はIPv4なので、sockaddr_inでよい。

これでIP addressは表示できそうだ。

## ホスト名はどうやって取得することができるのか？
### ホスト名の取得はgetnameinfo関数を用いて行う。

getnameinfo - アドレスから名前への変換をプロトコルに依存しないかたちで行う

int getnameinfo(const struct sockaddr *addr, socklen_t addrlen,
                       char *host, socklen_t hostlen,
                       char *serv, socklen_t servlen, int flags);

servはサービスが取得できますが、今回はホストの取得のみでいいので char *servとsocklen_t servlenは0に設定します。

```
#include <arpa/inet.h>
#include <netdb.h>
#include <netinet/in.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <strings.h>
#include <sys/socket.h>
#include <sys/types.h>
#include <time.h>
#include <unistd.h>

#define PORTNUM 13000
#define HOSTLEN 256
#define oops(msg)                                                              \
  {                                                                            \
    perror(msg);                                                               \
    exit(1);                                                                   \
  }
int setup_server(int port);

int main(int ac, char *av[]) {
  struct sockaddr_in saddr_cliant;
  socklen_t saddr_cliantlen;
  int sock_id, sock_fd;
  FILE *sock_fp;
  time_t thetime;

  sock_id = setup_server(PORTNUM);
  if (sock_id == -1) {
    perror("setup_server");
    exit(1);
  }
  while (1) {

    memset((void *)&saddr_cliant, 0,
           sizeof(saddr_cliant)); /*構造体をクリアする*/

    char buf[INET_ADDRSTRLEN];
    saddr_cliantlen = sizeof(saddr_cliant);

    sock_fd =
        accept(sock_id, (struct sockaddr *)&saddr_cliant, &saddr_cliantlen);

    if (sock_fd == -1)
      oops("accept");

    inet_ntop(AF_INET, &saddr_cliant.sin_addr, buf, sizeof(buf));

    char host[NI_MAXHOST];

    int ret = getnameinfo((struct sockaddr *)&saddr_cliant, saddr_cliantlen,
                          host, sizeof(host), 0, 0, 0);

    printf("Got a call ");
    if (ret == 0) {
      printf("from %s (%s)\n", buf, host);
    } else {
      printf("from %s \n", buf);
    }

    sock_fp = fdopen(sock_fd, "w");

    if (sock_fp == NULL)
      oops("fdopen");

    thetime = time(NULL);

    fprintf(sock_fp, "The time here is ...");
    fprintf(sock_fp, "%s", ctime(&thetime));
    fclose(sock_fp);
  }
}

int setup_server(int port) {
  struct sockaddr_in saddr;
  char hostname[HOSTLEN];
  int sock_id;

  /* ステップ1:カーネルにソケットを要求する。*/

  sock_id = socket(PF_INET, SOCK_STREAM, 0); // ソケット取得
  if (sock_id == -1)
    return -1;

  /* ステップ２：ソケットにアドレスをバインドする。アドレスはホスト、ポート*/

  memset((void *)&saddr, 0, sizeof(saddr)); /*構造体をクリアする*/

  saddr.sin_addr.s_addr = htonl(INADDR_ANY);
  saddr.sin_port = htons(
      PORTNUM); // htons()関数でリトルエンディアン→ビッグエンディアンに変換する。（ネットワークはビッグエンディアンが共通語になっている。）
  saddr.sin_family = AF_INET;

  if (bind(sock_id, (struct sockaddr *)&saddr, sizeof(saddr)) != 0)
    return -1;

  /*
   * ステップ3:キューサイズを１としてソケットを着信できる状態にする。
   */

  if (listen(sock_id, 1) != 0)
    return -1;

  return sock_id;
}
```

## まとめ
今回の課題で理解が重要なものは３つあった。
### sockaddr 構造体
ソケットの情報を詰める構造体だが、ソケットがどの種類の通信を行うかによって、実際の構造体を変える必要がある。
IPv4のインターネット通信なら、sockaddr_in構造体を使う。

### accept()関数の第２、第３引数
第２引数にsockaddr構造体、第３引数にその長さを入れる。
実際に渡したい構造体はsockaddr_in構造体のアドレスのため、(sockaddr *)にキャストして渡そう。
第３引数もsockaddr_in構造体の長さを取得して渡そう。

### getnameinfo関数
この関数でsockaddrからホスト名とサービス名を割り出すことができる。
今回はホスト名のみだったので、servポインタと、そのサイズは0に設定した。
取得したhost文字列を用い、表示を行うことで実現することができた。

### 注意点
ネットワーク内のローカルアドレス 192.168.0.6などはホスト名を逆引きできないため表示できないのは仕様である。
今回はif文でhostが取得できてなかった場合は(host)を表示しないという対応をしている。

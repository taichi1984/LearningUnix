# 研究課題10.15
## 課題内容
**teeの処理** teeユーティリティを使えば、データをファイルにリダイレクトするとともに、
データをパイプラインの次にコマンドに渡すこともできるようになる。例えば、次のコマンドは、
```
who | tee userlist | sort > list2
```
ソートされていないuserlistとソートされたlist2を生成する。
teeの引数は、ファイルの名前である。

詳しくはマニュアルを参照してほしい。
データをプログラムにリダイレクトするとともに、パイプラインの次のコマンドにも渡すprogteeというプログラムを書きなさい。
例えば次のパイプラインは
```
who | progtee mail smith | sort | progtee mail -s "hello" root > list2
```
ソートされていないリストをsmithにメールし、ソート済みのリストをルートにメールし、さらにソート済みリストをlist2ファイルに書き込む。

## 動作の予測
```
who | tee userlist | sort > list2
```
この１行において、
teeのやっていることは、標準入力から受け取ったものを引数にかかれたファイルに書き出すこと。
そして、標準出力に標準入力から受け取った結果をそのまま流すことだと思われる。


```
who | progtee mail smith | sort | progtee mail -s "hello" root > list2
```
progteeは何をしているのかというと標準入力からwhoを受取り
標準出力には受け取ったものをそのまま表示する。
引数で受け取ったコマンドに関しては、それをそのままパイプで引数のプロセスにつないで実行するという処理を行っていると思われる。

## まずはteeを自作して、２つの場所に書き込みをするイメージを掴む
### ソースコード
```
#include <fcntl.h>
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

ssize_t write_all(int fd, const char *buf, ssize_t n) {
  ssize_t total = 0;

  while (total < n) {
    ssize_t w = write(fd, buf + total, n - total);

    if (w == -1) {
      perror("write");
      return -1;
    }

    total += w;
  }
  return total;
}

int main(int ac, char *av[]) {
  char buf[BUFSIZ];
  int fd;
  if (ac != 2) {
    fprintf(stderr, "usage : progtee filename\n");
    exit(1);
  }

  if ((fd = open(av[1], (O_WRONLY | O_CREAT | O_TRUNC), 0644)) == -1) {
    perror("open");
    exit(1);
  }

  ssize_t n;
  while ((n = read(STDIN_FILENO, buf, sizeof(buf))) > 0) {

    if (write_all(fd, buf, n) == -1)
      exit(1);
    if (write_all(STDOUT_FILENO, buf, n) == -1)
      exit(1);
  }
  if (n == -1) {
    perror("read");
    exit(1);
  }

  close(fd);
}
```
標準入力からデータを読み込み、標準出力と、引数に指定されたファイルのfdを使って書き込む。
これで同時に２つの場所に出力していることになる。
writeの書き込みを行う際に、ファイルに書き込む際はあまり起こらないことだが、パイプなどに書き込む際は注意点がある。
それは、パイプに書き込んだ量を管理することである。
writeは、書き込み先に空きがあればその分だけ書き込みを行い、1バイトでも書き込めた場合はその時点で戻る。
書き込み先に全く空きがない場合のみ、ブロックして空きを待つ。
そのため、標準入力から読み込んだ分、総て書き込むようなコードを書いていると、読んだ分だけ実際は書けなかった場合に、ズレが発生することになる。
それに対処したのが以下の関数である。
```
ssize_t write_all(int fd, const char *buf, ssize_t n) {
  ssize_t total = 0;

  while (total < n) {
    ssize_t w = write(fd, buf + total, n - total);

    if (w == -1) {
      perror("write");
      return -1;
    }

    total += w;
  }
  return total;
}
```

totalで書き込んだ総量を管理し、実際にwriteを行う場合は、buf + total(書き込んだバイト数）にすることで、実際に書き込んだ分だけ書き込み開始の場所をずらす。
そして、書き込む量をn - totalにすることで、必ず書き込む量を n以上の値にならないように調整する（過剰な書き込みを防ぐ）
この処置をすることで、pipeに全体を一気に書き込むことができず分割書き込みが発生した場合でも、
残りのデータを正しく書き込まないとデータが欠損する可能性がある。
そのため、書き込んだバイト数を管理し、残りを繰り返し書き込む必要がある。

## teeプログラムを元にprogteeプログラムを書く
### ソースコード
```
#include <fcntl.h>
#include <stdio.h>
#include <stdlib.h>
#include <sys/wait.h>
#include <unistd.h>

ssize_t write_all(int fd, const char *buf, ssize_t n) {
  ssize_t total = 0;

  while (total < n) {
    ssize_t w = write(fd, buf + total, n - total);

    if (w == -1) {
      perror("write");
      return -1;
    }

    total += w;
  }
  return total;
}

int main(int ac, char *av[]) {
  char buf[BUFSIZ];
  int fd;
  int thepipe[2];
  ssize_t n;

  if (ac < 2) {
    fprintf(stderr, "usage: prog command\n");
    exit(1);
  }

  if ((pipe(thepipe)) == -1) {
    perror("pipe");
    exit(1);
  }
  if (fork() == 0) {
    dup2(thepipe[0], 0);
    close(thepipe[0]);
    close(thepipe[1]);
    execvp(av[1], &av[1]);
    perror("exec");
    exit(1);
  }

  close(thepipe[0]);
  while ((n = read(STDIN_FILENO, buf, sizeof(buf))) > 0) {
    if (write_all(STDOUT_FILENO, buf, n) == -1)
      exit(1);
    if (write_all(thepipe[1], buf, n) == -1)
      exit(1);
  }
  close(thepipe[1]);
  if (n == -1) {
    perror("read");
    exit(1);
  }
  wait(NULL);
}
```

標準出力を２箇所（pipeと端末）につなぐことはできないので、
読み込んだ内容を標準出力とpipe両方に書き込むという処理を行う。
pipeの接続とdup2による標準入力の切り替えを正しく行えば、
同じデータを複数の出力先に書き込むことで実現できる。

親がwaitするタイミングは子が必要とする親の処理が全て終わってからにすること。
子が必要なデータを親が子に渡す前に、子の終了を待ってしまうと子は親からのデータを待つのでデッドロックが発生する。
waitを書く実行するタイミングには注意すること。

### 実行結果
```
$ who
taichi84 seat0        2026-03-27 21:37 (login screen)
taichi84 :1           2026-03-27 21:37 (:1)

$ who | ./progtee wc | sort
      2      11      98
taichi84 :1           2026-03-27 21:37 (:1)
taichi84 seat0        2026-03-27 21:37 (login screen)
```

## まとめ
標準入力からのデータを、標準出力とpipeの両方につないで、標準出力への１回のwriteで両方に書き込むといったことはできない。
そのため、標準入力からとったデータをきちんと標準出力とpipeの両方へ書き込む必要がある。
pipeの用意とプロセスへのdup2の接続さえしっかりできてしまえば、難しいことはないが
pipeへのwriteによる書き込みは必ず指定した量書き込むことを保証せず、実際に分割書き込みになることが起こりうるため
戻り値を用いて実際に書き込まれたバイト数を確認し、残りを繰り返し書き込む処理が必要である。

# 研究課題10.04
## 課題内容
pipe.cでは、読み出し側のプロセスであるsortは、パイプの書き込み側のディスクリプタのコピーをクローズしている。読み出し側プロセスが、パイプの書き込み側デスクリプタをクローズしないようにコードを書き換えなさい。そして、プログラムを実行し動作を説明しなさい。

## pipe.cのソースコード
```
/* pipe.c
 * 目的：１つのプロセスから別のプロセスにデータを流す。
 * パイプラインの作り方を示す。引数として２つのコマンドを取り、av[1]の出力をav[2]の出力に接続する。
 *
 *  使い方 pipe command1 command2
 *  効果 : command 1 | command2
 *  制限 : コマンドは引数を取ることができない。
 *  引数の個数がわかっているのでexeclp()を使う。
 *  注意：子と親を取り替えると面白いことになる。
 */

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

#define oops(m, x)                                                             \
  {                                                                            \
    perror(m);                                                                 \
    exit(x);                                                                   \
  }

int main(int ac, char **av) {
  int thepipe[2], /*2つのファイルディスクリプタ*/
      newfd, pid;

  if (ac != 3) {
    fprintf(stderr, "usage:pipe cmd1 cmd2\n");
    exit(1);
  }
  if (pipe(thepipe) == -1)
    oops("Cannot get a pipe", 1);

  /* ----------------------------------------------------------
   * パイプを作ったのでプロセスを２つにする */

  if ((pid = fork()) == -1)
    oops("Cannot fork", 2);

  /* -----------------------------------------------------------
   * この時点でプロセスは２個ある。親はパイプから読み出す*/

  if (pid > 0) {       /*親はav[2]を実行する*/
    close(thepipe[1]); /*親はパイプに書き込まない*/

    if (dup2(thepipe[0], 0) == -1)
      oops("could not redirect stdin", 3);

    close(thepipe[0]); /*標準入力の複製を作り、パイプをクローズする。*/
    execlp(av[2], av[2], NULL);
    oops(av[2], 4);
  }

  /* 子はav[1]を実行し、パイプに書き込む*/
  if (pid == 0) {
    close(thepipe[0]); /*子はパイプから読み出さない*/

    if (dup2(thepipe[1], 1) == -1)
      oops("could not redirect stdout", 4);

    close(thepipe[1]); /*標準出力の複製を作り、パイプをクローズする。*/

    execlp(av[1], av[1], NULL);
    oops(av[1], 5);
  }
}
```

以上のソースコードでは
```
if(pid > 0){
    close(thepipe[1]);

```
というようにpipeの書き込み側をクローズしている。
この一行をコメントアウトすればいいだろう。

## なぜブロックして入力待ちになるのか？
### pipeと処理の流れを追う。
```
  int thepipe[2], /*2つのファイルディスクリプタ*/
      newfd, pid;

  if (ac != 3) {
    fprintf(stderr, "usage:pipe cmd1 cmd2\n");
    exit(1);
  }
  if (pipe(thepipe) == -1)
    oops("Cannot get a pipe", 1);

  /* ----------------------------------------------------------
   * パイプを作ったのでプロセスを２つにする */

  if ((pid = fork()) == -1)
    oops("Cannot fork", 2);

```

パイプを作りforkでプロセスを複製するコードである。

pipeコマンドでパイプを作ったあとにforkをすることで複製をする。
そうするとパイプへの接続するためのfdであるthepipe[0],thepipe[1]も子プロセスで複製される。

現在の接続先の状態は
親プロセス　→　パイプへの読み取りの接続と書き込みの接続を両方持つ
子プロセス　→　パイプへの読み取りの接続と書き込みの接続を両方持つ

状態になる。

子プロセス側の読み取りの接続を閉じ、親プロセスの書き込みを閉じなかった場合は以下の状態に接続が変わる。

親プロセス　→　パイプへの読み取りの接続と書き込みの接続を両方持つ
子プロセス　→　パイプへの書き込むための接続を持つ

パイプは書き込み側にアクセスを持つすべてのプロセスがパイプの書き込み側をクローズすると、パイプからのreadは０を返す。
そのため、親プロセス側の書き込みを閉じないと、親プロセス側の読み込みは、まだ入力があるものと思い込み、入力をまち続けてしまう。
そのため、書き込み側もクローズしないと、いつまでもブロックされた状態でまち続けることになる。

このため、不要な書き込みfdはクローズしておかないと、読み込みが終了しなくなってしまう。


## 結論
以上の動作から書き込み側のファイルディスクリプタは使わないものは必ずcloseする必要がある。
複製した際に使わないファイルディスクリプタを閉じるのと、dup2したあとの複製元のファイルディスクリプタを閉じるのを忘れないようにすること。

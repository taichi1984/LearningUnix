# 研究課題10.09
## 課題内容
dupはlinkとどのような点が似ているか。ポインタについて論じなさい。

## dup()は何をするのか？
int dup(int oldfd);

oldfdと同じファイルを指す新しいファイルディスクリプタを作ります。
新しいファイルディスクリプタは一番若い番号が割り当てられます。


実際はこれだと意図しないファイルディスクリプタに複製したものが割り当てられる可能性があるので
dup2(oldfd,newfd)関数を使って、複製したいfdと新たに作りたいfdを引数に指定することで、意図しない接続先になることを防ぎます。
newfdがすでにどこかに割り当てられている場合はそのfdをクローズしてからそこに新たなfdを作ります。



## link()は何をするのか？
int link(const char *oldpath , const char *newpath);

oldpathのファイルにnewpathという新しい名前のハードリンクを作成する。

## どのような点が似ているか
あるデータに対してのポインタを複製するという点で酷似しています。

dupはファイルへの接続であるファイルディスクリプタを複製します。
これはファイルのinodeへのリンクを増やすlink()と非常に似たようなことをしています。
なぜなら、dupで複製するファイルディスクリプタはinodeに接続するためのopen file tableへのリンクを複製しており
link()は特定のinodeのデータにアクセスするためのリンクをファイルという形で複製しているからです。

ファイルディスクリプタはあくまでファイル操作をする仕組みへのリンクであり、
link()で複製されたハードリンクはinodeのデータへのリンクそのものなので、複製している対象は違いますが
動作としてはかなり酷似しているものになります。

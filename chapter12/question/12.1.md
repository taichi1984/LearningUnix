# 研究課題
## 課題内容
時報クライアントのサンプルは、readとwriteを一度ずつ呼び出す。しかし、サーバからのデータが数回に分けて届いたり、
バッファサイズよりも大きかったりした場合にはどうすればよいのだろうか。バッファサイズよりも大きかったりした場合にはどうすればよいのだろうか。
複数回のread呼び出しが必要なデータを処理できるようにクライアントを書き換えるにはどうすればよいだろうか。
write呼び出しで文字列の長さより小さい値が返されたら、サーバはどうすればよいか。

## readとwriteを必要な回数分行える関数を作る。
readもしくはwriteはreadするデータ量やwriteするデータの量を指定できるが、
実際にその量がreadまたはwriteされるかはわからない。
関数の戻り値が実際に読み込んだ（書き込んだ）バイト数、であることを使って
write_allやread_allという関数を自作し、期待したデータ量を確実に読み込んだり書き込んだりできるようにする必要がある。

### write_all()のコード

```
ssize_t write_all(int sock_fd, const void *buf, size_t len) {
  ssize_t wr = -1;
  ssize_t total = 0;
  const char *p = buf;

  while (total < len) {
    wr = write(sock_fd, p + total, len - total);

    if (wr < 0) {
      return -1;
    }
    if (wr == 0) {
      break;
    }
    total += wr;
  }
  return total;
}
```

total < lenの間writeを続けるコード
これでlenで指定された長さ分、確実にwriteすることが可能。
writeはバッファの都合で一部しか送れないことがある。特にソケットは送信バッファがあるため、
送れなかった分は送る必要がある。

### read_all()のコード
```
ssize_t read_all(int fd, void *buf, size_t len) {
  ssize_t total = 0;
  ssize_t rn;
  char *p = buf;
  while (total < len) {
    rn = read(fd, p + total, len - total);
    if (rn < 0) {
      return -1;
    }
    if (rn == 0) {
      break;
    }
    total += rn;
  }

  return total;
}
```

totalがlenになるまで読み込みを試みるが、
readが0を返した場合（接続終了）やエラーが発生した場合には、
その時点までに読み込めたデータを返す。

したがって、この関数は指定したバイト数を必ず読み込めることを保証するものではなく、
あらかじめデータ長が分かっている場合にのみ有効である。

## まとめ
read,writeの戻り値である、実際の読み込み（書き込み）データ量を使って管理を行うことにより
必要な量だけ確実に読み書きをすることが可能。

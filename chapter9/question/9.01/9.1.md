# 研究課題9.1
## 課題内容
**組み込みコマンドの危険性** setという名前のCプログラムかシェルスクリプトを使って、シェルからそのプログラムの起動を試みなさい。
何が起きるか。no=diceという名前のCプログラム可シェルスクリプトを使って、シェルからそのプログラムの起動を試みなさい。何が起きるか。
testという名前のプログラムで同じことをしなさい。これらのプログラムを起動する方法はあるか。

## set プログラムとtestプログラムとno=dice.cプログラムを用意
各プログラムの内容は

printf("this is set program\n");

というように１行だけ自分がなにプログラムか表示するだけのプログラムである。

## 実際にやってみる
```
$ set
各種変数や関数がたくさん表示される(内容は略)

$ test
何も表示されない

$ no=dice.c
$ echo $no
dice.c

```

自分が作ったプログラムより組み込みコマンドやshellの代入構文のほうが優先されているようだ。

## これらのプログラムを実行する方法について
今の自分の環境のシェルはbashだが、
現在のbashはカレントディレクトリにある実行可能プログラムは明示的にカレントディレクトリを指定しない場合、カレントディレクトリを探索しないようになっている。
実行したい場合は

./set
./test
./no=dice.c

などとプログラム名を指定する必要がある。
これは組み込みコマンドとの混同を避ける用途も兼ねており、かなりいい仕様になっているものと思われる。

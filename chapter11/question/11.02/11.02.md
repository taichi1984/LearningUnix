# 研究課題11.02
## 課題内容
**popenとシグナル** 本文で作ったpopenは、シグナルに対して何もしないがそれでよいのか。
子は、親からシグナル処理設定を継承する。親が終了、無視、関数呼び出しの３種類の処理方法のどれを選んでいるかによって、３通りの答えを用意しなさい。

## 課題で作ったpopen.c
### コード
```
/* popen.c - UNIXライブラリ関数のpopen()の実装の１つ
 * FILE *popen(char *command , char *mode)
 * command は通常のシェルコマンド
 * modeは"r"か"w"
 * コマンドに接続されたストリームかNULを返す。
 * execlで"sh" "-c" command を実行する。
 * 課題：子プロセスのシグナル処理をどうするか。
 */

#include <signal.h>
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

#define READ 0
#define WRITE 1

FILE *popen(const char *command, const char *mode) {
  int pfp[2], pid;
  FILE *fp;
  int parent_end, child_end;

  if (*mode == 'r') {
    parent_end = READ;
    child_end = WRITE;
  } else if (*mode == 'w') {
    parent_end = WRITE;
    child_end = READ;
  } else
    return NULL;

  if (pipe(pfp) == -1)
    return NULL;
  if ((pid = fork()) == -1) {
    close(pfp[0]);
    close(pfp[1]);
    return NULL;
  }

  /* ------------------- 親のコード -------------*/
  /* ------------------- 片方をクローズしてもう片方をfdopenする。*/

  if (pid > 0) {
    if (close(pfp[child_end]) == -1)
      return NULL;
    return fdopen(pfp[parent_end], mode);
  }

  /* -------------------子のコード ------------------*/
  /* stdin またはstdoutをリダイレクトし、cmdをexec */
  if (close(pfp[parent_end]) == -1)
    exit(1);

  if (dup2(pfp[child_end], child_end) == -1)
    exit(1);

  if (close(pfp[child_end] == -1))
    exit(1);

  execl("/bin/sh", "sh", "-c", command, NULL);
  exit(1);
}
```

## シグナル引き継ぎのルールの復習。
exec後のシグナルに関しては以下のルールになっている。

1,ユーザー定義ハンドラはリセット
2,デフォルトと無視は引き継がれる

今回のケースは終了、無視、ハンドラの３パターンについての話になってくる。

## どうすべきか。
### ユーザー定義ハンドラ
execにより自動的にSIG_DFLに戻るため、特別な処理は不要。

### デフォルトを設定してたとき、SIG_DFL設定時
デフォルト設定にしてたならそのままで良い。


### 無視の設定をしていた場合、SIG_IGN設定時
execl後まで、シグナルの処理を引き継いでしまうので、デフォルトに戻ると思いこんでいると想定外の挙動になる危険がある。
SIG_IGNの設定は引き継がれるということは、認識しておく必要がある。
必要に応じて、シグナル状態を初期化することが望ましい。


## シグナルマスクについて
シグナルマスクの設定に関しては引き継がれるので注意が必要。
SIG_DFLに戻ると認識していても、SIGTERMを弾く設定をマスクでしていたため、SIGTERMを終了処理できなくなるとかはありうる。
マスクを使っている場合は、シグナルマスクも全て外してしまったほうが無難だろう。
